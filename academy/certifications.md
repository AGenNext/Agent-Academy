# Certification Ladder

AGenNext certifications validate production capability. They are not badges for watching lessons. Every level requires evidence.

## Certification Levels

| Level | Certification | Focus |
|---:|---|---|
| 100 | Agent Practitioner | responsible usage and task framing |
| 200 | Agent Engineer | building agent workflows and integrations |
| 300 | Senior Agent Engineer | production-grade systems and governance |
| 400 | Agent Architect | enterprise architecture and design authority |
| 500 | Enterprise Agent Platform Architect | organization-scale platform design |
| 600 | Certified AGenNext Operator | operating and governing production systems |

---

## Level 100 — Agent Practitioner

### Candidate Can

- Write clear task briefs
- Use AI tools responsibly
- Verify output against evidence
- Identify unsafe automation
- Escalate risky actions

### Required Evidence

- Task brief
- Lab artifact
- Review note
- Safety reflection

---

## Level 200 — Agent Engineer

### Candidate Can

- Build agent workflows
- Use Claude Code or equivalent coding agents safely
- Build MCP or tool integrations
- Package reusable Skills
- Create evaluation cases

### Required Evidence

- Working lab
- Tool contract
- Skill artifact
- Evaluation set
- PR or change plan

---

## Level 300 — Senior Agent Engineer

### Candidate Can

- Build governed multi-agent systems
- Design subagent roles and boundaries
- Add hooks and policy gates
- Build audit and evidence trails
- Prepare release-ready agent workflows

### Required Evidence

- Multi-agent workflow design
- Governance matrix
- Hook policy
- Evaluation report
- Production readiness review

---

## Level 400 — Agent Architect

### Candidate Can

- Design agent platform architecture
- Define runtime, memory, identity, tool, and policy boundaries
- Compare vendor options
- Define safe integration patterns
- Lead architecture reviews

### Required Evidence

- Reference architecture
- Threat model
- Tool boundary model
- Platform decision record
- Review checklist

---

## Level 500 — Enterprise Agent Platform Architect

### Candidate Can

- Design organization-wide agent platforms
- Define governance and operating models
- Map compliance obligations to platform controls
- Design certification and enablement programs
- Establish vendor-neutral architecture

### Required Evidence

- Enterprise platform blueprint
- Governance standard
- Operating model
- Capability map
- Rollout plan

---

## Level 600 — Certified AGenNext Operator

### Candidate Can

- Operate production agent systems
- Monitor health, trust, cost, and incidents
- Approve or block risky changes
- Execute runbooks
- Produce operational evidence

### Required Evidence

- Operator runbook
- Incident simulation
- Release approval decision
- Cost and trust report
- Post-incident review

## Certification Decision Model

Each certification review ends with one of three decisions:

| Decision | Meaning |
|---|---|
| Certified | all critical evidence is present |
| Certified with Conditions | minor gaps remain but capability is demonstrated |
| Not Certified Yet | missing evidence or unsafe design |

## Evidence Rule

No evidence, no certification.

Claims must be backed by files, commands, logs, tests, screenshots, diagrams, pull requests, or review notes.
